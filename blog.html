What it does

A stop codon is the three-letter signal that tells the ribosome to end a protein. In the standard code there are three of them — TAA (ochre), TAG (amber), TGA (opal) — though some organisms use a different set. The Stop Codon Finder reads a DNA sequence and reports every stop codon in all six reading frames (three forward, three reverse), with exact coordinates.

Drop in a plain FASTA sequence and you get the full six-frame catalogue. Drop in an annotated genome — a GenBank file, a GFF3, or a FASTA + GFF3 pair — and it goes further: it tells you, for each stop codon, which gene it lands in and what role it plays.

Why it's different

Plenty of tools list open reading frames (NCBI ORFfinder, EMBOSS, ExPASy Translate, the Sequence Manipulation Suite). What none of the free ones do is put each stop codon in the context of a genome annotation. This tool classifies every hit:

ContextWhat it means
cds-terminatorThe stop that properly ends a gene.
cds-internal-inframeAn in-frame stop inside a gene — a candidate for readthrough, selenocysteine (Sec), a programmed frameshift, a pseudogene, or a sequencing error.
cds-internal-outframeA stop inside a gene but in a different frame.
intergenicOutside any annotated feature.

On top of that it handles things most web tools skip: circular genomes (stop codons that straddle the origin of a phage or plasmid), the full set of NCBI genetic-code tables (so TGA is correctly read as tryptophan in Mycoplasma, or TCA/TTA as stops in the codes that use them), IUPAC ambiguity codes, and a sanity check that warns you when the genetic code you picked looks wrong for your data.

How to use it

  1. Load your sequence Drag a file onto the drop zone or click Choose File(s). Accepted: FASTA (.fasta/.fa/.fna), GenBank (.gb/.gbk), GFF3 (.gff3), or a FASTA + GFF3 pair. No file handy? Pick a bundled example from Load example — including a real Lambda phage genome.
  2. Check the settings (optional) The tool auto-detects the genetic code and turns on annotation-aware mode when it finds features. You can override the genetic code / translation table, indicate your organism, or switch between all stop codons and coding/predicted mode.
  3. Run the scan Click Run scan. A progress bar shows real throughput and an estimated time remaining — even multi-megabase genomes stay responsive.
  4. Read the results You get summary stats (counts per codon, frame and strand, GC%, density), a six-frame map and density plot, a per-gene summary, and a sortable, filterable table of every hit with its gene and context.
  5. Export Download CSV, JSON, GFF3, BED, FASTA of ORFs/proteins, or the charts as SVG/PNG. Every hit keeps one stable ID across all formats, so the files cross-reference cleanly and drop straight into IGV or a genome browser.

Tip: keep the tab focused for very large genomes — browsers slow down background tabs, though your results are never affected.

The science, briefly

Six reading frames

DNA is double-stranded, so a sequence can be read six ways: forward frames +1/+2/+3 (starting at the 1st, 2nd, or 3rd base), and reverse frames −1/−2/−3 (the same, on the reverse complement). A stop codon in any of these frames is reported.

Coordinates you can trust

Every hit is reported in 1-based, inclusive coordinates on the forward strand — the same convention as GFF3 and GenBank — so a start/end pair always points at the exact bases in your file, whichever strand the codon was found on. The BED export converts to the 0-based, half-open convention that format expects; the app spells out the difference so nothing is ambiguous.

Not every in-frame stop is a mistake

A stop codon inside a gene isn't always an error. It can be recoded — selenocysteine reads through TGA, and some genes use programmed frameshifts. These can't be proven from sequence alone, so the tool flags them as candidates rather than calling them broken — and honors GenBank /transl_except so real Sec/Pyl recodings aren't mislabeled.

Host your own copy (free, in minutes)

The whole tool is static files — no server, no build. The easiest home is GitHub Pages:

  1. Put the files in a repo Create a repository (e.g. stop-codon-finder) and add the project files (index.html, css/, js/, blog.html).
  2. Push it 
    git add .
git commit -m "Stop Codon Finder"
git branch -M main
git remote add origin https://github.com/<you>/stop-codon-finder.git
git push -u origin main
  3. Turn on Pages In the repo, go to Settings → Pages, set Source to Deploy from a branch, choose main and / (root), and save.
  4. Share the link Your tool goes live at https://<you>.github.io/stop-codon-finder/, and this guide at …/blog.html.

You can also just double-click index.html to run it straight from your computer, offline.

Privacy & accessibility

Your data stays on your machine. Files are read locally in the browser and no network request is made after the page loads — you can verify this in the browser's Network tab. That matters for unpublished sequences.

The interface works in light and dark themes, meets WCAG AA contrast, is keyboard-operable, uses a colorblind-safe codon palette, and ships with a built-in Learn section, glossary, and inline help so it's approachable for students and rigorous enough for researchers.
